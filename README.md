### [Version 1.0.]

Testbed based on the [SimpleFrequencyDetector](https://github.com/ArminJo/Arduino-FrequencyDetector) library.

And that is the background story:

From Thursday 27.2.2020 21:00 CET out of the blue my Travis-CI account suddenly showed me **"build has errored" on every new commit**.
In the log file I could see *"Timeout waiting for network availability"*  the username was [redacted].
With this account, however, I could see my old builds, which could all be restarted successfully.<br/>
After my support request the same evening, I only got a query, but despite answering, nothing happend.
The next day, after further inquiries (10:00 am + 1:00 pm), someone from America answerd at 2:30 pm (8:30 am EST) with **"we are sorry about the hassle"**.
Fine!<br/>
But in the support email it said: *"I think you have made some changes to GitHub handle for the account [redacted]? ... Could you please confirm that you have made changes to account name?"*
After my answer *"I am sorry but I NEVER made any changes to my account. Not on Github nor at travis!!!"* **nothing happend**.<br/>
It seems that, if you don't pay money, you have to live with the fact that your account can be lost (forever?) :-(<br/>
So I invested 2 days to switch the CI to **GitHub Actions** and found that GitHub Actions run about **5 times faster** and of course are better integrated.

Mein Travis-CI Account hat mir ab Donnertag den 27.2.2020 21:00 CET aus heiterm Himmel **bei jedem neuen Commit** plötzlich nur noch **"build has errored"** angezeigt.
Im Logfile war "Timeout waiting for network availability" zu sehen, als Username [redacted].
Unter dem Account konnte ich aber alle alten builds von mir sehen, die sich auch alle erfolgreich restarten liessen.<br/>
Nach meiner Support Anfrage an selben Abend bekam ich erst eine Rückfrage, aber trotz deren Beantwortung regte sich nichts mehr.
Am nächsten Tag meldete sich erst nach weiteren Anfragen (10:00 + 13:00) um 14:30 jemand aus Amerika, dort war es 8:30, mit **"we are sorry about the hassle"**.
So weit so schön.<br/>
Aber in der Supportmail stand sinngemäss drin: *"Wir denken, sie haben an dem GitHub handle Ihres accounts [redacted] etwas geändert. Können sie das bestätigen?"*
Nachden ich zurückgechrieben habe, dass ich nichts geändert habe -woher soll ich auch einen [redacted] account herbekommen und was ist der "GitHub handle"???- tat sich nichts mehr.<br/>
Wenn man kein Geld bezahlt muss man wohl damit leben, dass einem der Account ersatzlos abhanden kommen kann :-(<br/>
Daher habe ich 2 Tage investiert, das CI auf **GitHub Actions** umzustellen und festgestellt, das es dort ca. **5 mal schneller** durchläuft und natürlich auch besser integriert ist.
